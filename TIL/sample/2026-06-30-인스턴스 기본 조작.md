# 2026-06-30 인스턴스 기본 조작

## 오늘 배운 내용

- 모든 클래스는 Object 클래스의 메서드를 가지고 있다.
- Object 타입 변수에는 모든 인스턴스를 대입할 수 있다.
- <Object 클래스의 대표 메서드>는 아래와 같다.
- ① toString() : 문자열 표현을 얻음.
- ② equals() : 비교.
- ③ hashCode() : 해시값을 얻음.
- 만든 클래스로 객체를 생성한 뒤, 그 객체를 그대로 출력(System.out.println();)하면 기본적으로는 알 수 없는 클래스 이름과 메모리 주소(해시코드)가 출력된다.
- 이러한 문제를 해결하기 위해서, toString() 메서드를 사용하면 사용자가 원하는 방식으로 데이터를 보여주도록 덮어쓰기(오버라이드)가 되어 이러한 문제가 해결된다.
- 자바에서 List와 Set은 데이터를 한데 모아 관리하는 컬렉션 프레임워크(Collection Framework)의 가장 핵심적인 두 가지 인터페이스(표준 규격)이다.
- List는 데이터를 일렬로 늘어놓은 선형 구조이다.
- 예를 들면, 대기자 명단이나 영수증 품목 목록을 생각하면 된다.
- 특징은 크게 두 가지인데, 첫 번째는 순서유지로 데이터를 넣은 순서가 그대로 유지되고 가장 먼저 넣은 데이터가 0번 인덱스(방 번호)에 위치한다는 것이다.
- 두 번째 특징은, 중복 허용이 되어서 똑같은 데이터를 여러 번 넣을 수 있다는 것이다.
- 대표적인 구현 클래스는 ArrayList, LinkedList, Vector가 있다.
- 사용 예시로는 음악 재생 목록(플레이리스트), 게시판 게시글 목록이 있다.
- Set은 수학에서 말하는 집합의 개념이다.
- 순서와 상관없이 주머니에 데이터를 마구 집어넣는 구조를 생각하면 된다.
- Set의 특징은 크게 두 가지인데, 첫 번째 특징은 순서가 없어서 데이터를 넣은 순서를 보장하지 않는다는 것이다.
- 이것은 내부적인 알고리즘에 의해 데이터가 배치되므로, 몇 번째로 넣었는지 알 수 없고 그래서 인덱스가 없다.
- 두 번째 특징은 중복 불가로 이미 존재하는 데이터를 또 넣으려고 하면 무시된다는 특징이다.
- 대표적인 구현 클래스는 HashSet, TreeSet, LinkedHashSet(이것은 Set이지만 예외적으로 순서를 유지해 준다.)이 있다.
- 사용 예시로는 로또 번호 추첨(중복 번호 없음), 회원 아이디 모음(중복 아이디 방지)이 있다.
- 데이터 검색은 List는 인덱스(get(i))를 이용해 빠르게 찾지만, Set은 인덱스가 없어서 원소 하나를 콕 집어서 가져오기 어렵다.(그래서 반복자나 stream을 사용한다.)
- equals() 메서드를 오버라이드하여 동등성 규칙을 정의할 수 있다.
- 이것은 List에서 동등성 비교시 사용된다.
- hashCode를 재정의하면 Set, Map 내부에서의 동등성 규칙으로 사용된다.
- Set, Map 계열은 요소를 검색할 때 equals()보다 비용이 싼 hashCode() 비교를 사용한다.
- List는 순차 검색이라 느리다.
- 모든 객체는 해시값을 가진다.
- 동일한 객체는 항상 같은 해시값을 가진다.
- 하지만 같은 해시값이라고 항상 동일한 객체는 아니다.
- Collections.sort() 메서드는 컬렉션 내부를 정렬해준다.
- Collections.sort() 메서드의 선언은 아래와 같다.
- public static <T extends Comparable<? super T>> void sort(List<T> list)
- 자바에서 Comparable 인터페이스는 객체들은 정렬(Sort)할 때 기준을 세워준다.
- 정렬 규칙을 정하기 위한 Comparable 인터페이스는 아래와 같다.
- public interface Comparable<T> {
  public int compareTo(@NotNull T o);
  }
- compareTo의 작동 원리(반환값의 의미)는 메서드를 오버라이딩할 때, 내 객체(기준)와 상대방 객체(o)를 비교해서 세 가지 중 하나의 값을 남기는 원리이다.
- 세 가지 반환값은 아래와 같다.
- ① 음수(주로 -1): 내가 상대방보다 작다는 뜻-> 순서가 앞으로 감.
- ② 0: 나와 상대방이 같다는 뜻.
- ③ 양수 (주로 1): 내가 상대방보다 크다는 뜻-> 순서가 뒤로 밀림.
- Comparable 인터페이스를 구현하지 않은 객체도 즉석에서 정렬 규칙을 정해줄 수 있다.
- 얕은 복사(Shallow Copy)는 객체 자체는 새로 생성되지만, 객체 내의 참조 타입 필드(다른 객체에 대한 참조)는 원본과 동일한 객체를 참조한다.
- 즉, 중첩된 객체는 복사되지 않고 주소만 복사된다.
- 원본 객체의 참조 타입 필드를 변경하면 복사본의 해당 필드도 함께 변경된다.
- 비유하면, 하나의 방에 들어갈 수 있는 열쇠를 하나 더 복사해 준 것과 같다.
- 깊은 복사(Deep Copy)는 객체를 복사할 때, 기존 객체와 동일한 값을 가진 새로운 객체를 메모리에 완전히 새로 생성한다.
- 장점은 원본 객체와 복사본 객체는 주소값이 완전히 다른 별개의 객체라서, 복사본을 수정해도 원본에는 아무런 영향을 주지 않는다는 장점이 있다.
- 비유하면, 기존에 있던 방과 똑같은 인테리어를 가진 새로운 방을 다른 위치에 하나 더 지어준 것과 같다.
- 이것은 새로 지은 방의 가구를 바꿔도 원래 방은 그대로 유지되는 것과 같다.

## 기억할 것

- Object 클래스의 대표 메서드 3가지가 언제 사용되는지 기억한다.
- List와 Set의 뜻, 특징, 대표적인 구현 클래스 그리고 사용 예시를 기억한다.
- equals() 메서드, hashCode, Collections.sort() 메서드, compareTo의 작동 원리(반환값의 의미)를 기억한다.
- 얕은 복사(Shallow Copy)와 깊은 복사(Deep Copy)의 각각의 뜻과 특징 그리고 비유를 기억한다.

## 실습 코드

```java
import java.util.Date;
import java.util.Objects;

public class Book implements Cloneable, Comparable<Book> {
    private String title;
    private Date publishDate;
    private String comment;

    public String getTitle() {
        return title;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public Date getPublishDate() {
        return publishDate;
    }

    public void setPublishDate(Date publishDate) {
        this.publishDate = publishDate;
    }

    public String getComment() {
        return comment;
    }

    public void setComment(String comment) {
        this.comment = comment;
    }

    @Override
    public final boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Book book)) return false;
        return Objects.equals(title, book.title) && Objects.equals(publishDate, book.publishDate);
    }

    @Override
    public int hashCode() {
        int result = Objects.hashCode(title);
        result = 31 * result + Objects.hashCode(publishDate);
        return result;
    }

    @Override
    public int compareTo(Book other) {
        return other.publishDate.compareTo(this.publishDate);
    }

    @Override
    public Book clone() {
        Book result = new Book();
        result.title = this.title;
        result.publishDate = (Date) this.publishDate.clone();
        result.comment = this.comment;
        return result;
    }
}
```

## 어려웠던 점

- (System.out.println();)하면 기본적으로는 알 수 없는 클래스 이름과 메모리 주소(해시코드)가 출력되는게 처음에는 이해가 어려웠다.
- "같은 해시값이라고 항상 동일한 객체는 아니다."가 이해하기 어려웠다.
- Collections.sort() 메서드의 선언이 익숙하지 않아서 직접 선언하는 코드를 외우고 쓰기 어려웠다.

## 해결 방법

- 코드가 출력되는 과정을 찾아보고 이해했다.
- 해시값에 대해서 자세히 찾아보고 이해했다.
- Collections.sort() 메서드의 선언할 때 쓰이는 코드를 무작정 외우지 않고, 코드에 담긴 의미를 하나씩 이해했다.
