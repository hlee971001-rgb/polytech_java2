# 2026-06-29 제네릭과 열거형

## 오늘 배운 내용

- 타입이 없으면 런타임 에러가 나기 쉽고, IDE가 컴파일 에러를 미리 찾을 수 없다.
- 제네릭(Generic)은 자바에서 데이터의 타입을 외부에서 지정할 수 있도록 일반화하는 기능이다.
- 다시 말하면, 클래스나 메서드를 만들 때 특정 타입을 미리 고정해두지 않고, 나중에 쓸 때(객체를 생성하거나 메서드를 호출할 때)정하고 일단 임시 기호로 비워두는 틀이다.
- 제네릭이 없을 때는 여러 타입을 담기 위해 모든 클래스의 최상위 부모인 Object타입을 사용했다.
- 하지만 Object타입의 문제점은 꺼낼 때 반드시 형변환(Casting)을 해야 해서 실수로 다른 타입을 넣으면 컴파일은 성공하지만 실행(Runtime)중에 프로그램이 튕기는 문제점이 있다.
- 제네릭의 핵심 장점 3가지 중 첫 번째는, 타입 안정성(Type Safety)으로 의도하지 않은 타입의 객체가 저장되는 것을 컴파일 시점에 완벽히 차단해 실행 중에 앱이 멈추는 불상사를 막아주는 것이다.
- 두 번째는, 불필요한 형변환 제거로 값을 꺼낼 때마다 타입을 강제로 변환해 줄 필요가 없어서 코드가 깔끔해진다.
- 마지막으로는, 코드 재사용성 증가로 하나의 클래스만 만들어두면 부품처럼 다양한 타입으로 재활용할 수 있다.
- 제네릭에서 자주 쓰는 암묵적 기호들이 있다. 기호 자체는 아무 알파벳이나 써도 상관없지만, 전 세계 개발자들이 관례적으로 사용하는 규칙이 있다.
- ① <T> : Type (가장 흔하게 사용되는 타입 일반)
- ② <E> : Element (ArrayList 등 컬렉션 내부의 요소를 뜻할 때)
- ③ <K> : Key (Map의 키)
- ④ <V> : Value (Map의 값)
- ⑤ <N> : Number (숫자 관련 타입)
- 일반 클래스 뒤의 extends: 부모의 능력을 그대로 물려받아 내 클래스를 확장(상속)한다.
- 제네릭 안의 extends: 이 주머니에 넣을 수 있는 데이터 타입을 부모 클래스의 자식 계열로만 제한한다.
- 열거형(Enum, Enumeration)은 자바에서 서로 연관된 상수의 집합을 저장하는 특별한 데이터 타입이다.
- 쉽게 말해, 딱 정해진 몇 가지 선택지만 존재하고 그 외의 값은 절대 들어오면 안 될 때, 그 선택지들을 안전하게 묶어서 관리하는 기능이다.
- 열거형의 문제점은 예를 들어 int타입일 때, 아무 숫자나 다 들어갈 수 있어서 문법적으로 에러 없이 작동한다는 것이 문제점이다.
- 열거형의 핵심 장점 3가지 중 첫 번째는, 타입 안정성으로 컴파일러가 허용된 선택지 외의 다른 값이 들어오는 것을 완벽하게 차단해 줌으로써 엉뚱한 값이 들어가서 발생하는 버그를 원천 봉쇄한다.
- 두 번째는, 코드의 가독성 향상으로 숫자 1, 2 같은 의미 없는 숫자(Magic Number) 대신 Job.WARRIOR처럼 직관적인 이름을 사용하여 코드가 무엇을 뜻하는지 바로 읽히는 게 장점이다.
- 마지막으로는, 인텔리제이의 자동 완성 지원으로 코딩할 때 Job.까지만 입력해도 사용 가능한 선택지 리스트(WARRIOR, WIZARD...)가 자동으로 화면에 떠서 오타 칠 일이 없다.
- AuthState는 인증상태라는 뜻이다.
- 익명클래스(anonymous class)는 정식으로 클래스 이름을 짓고 상속받는 과정을 전부 생략하고, 메서드에 값을 넘겨주는 순간에 {}을 열어서 넘겨주는 문법이다.
- 이름이 없기 때문에 그 자리에서만 쓰이고 사라지는 일회용 소모품 클래스 같은 개념이다.

## 기억할 것

- 제네릭의 뜻과 사용하는 이유, 핵심 장점 3가지, 제네릭에서 자주 쓰는 암묵적 기호들을 기억한다.
- 열거형의 뜻과 사용하는 이유, 핵심 장점 3가지를 기억한다.
- 익명클래스의 뜻을 기억한다.

## 실습 코드

```java
public class StrongBox<T> {
    private T item;

    public void put(T item) {
        this.item = item;
    }

    public T get() {
        return this.item;
    }
}
```

```java
enum KeyType {
    PADLOCK(1024),
    BUTTON(10000),
    DIAL(30000),
    FINGER(1000000);

    private final int limit;

    KeyType(int limit) {
        this.limit = limit;
    }

    public int getLimit() {
        return this.limit;
    }
}

public class StrongBox2<T> {
    private T item;
    private final KeyType keyType;
    private int count;

    public StrongBox2(KeyType keyType) {
        this.keyType = keyType;
        this.count = 0;
    }

    public void put(T item) {
        this.item = item;
    }

    public T get() {
        this.count++;

        if (this.count < this.keyType.getLimit()) {
            return null;
        }

        return this.item;
    }
}
```

## 어려웠던 점

- 일반 클래스 뒤의 extends와 제네릭 안의 extends가 각각 쓰임이 다른게 어려웠다.
- Object타입의 문제점이 발생할 때, 일어나는 현상이 이해하기 어려웠다.
- 익명클래스가 헷갈렸다.

## 해결 방법

- 일반 클래스와 제네릭의 차이점을 정확히 이해하고 다시 보니깐 이해가 쉬워졌다.
- Object타입의 문제점이 발생할 때, 일어나는 현상을 컴퓨터 작동 측면에서 이해하니 이해가 쉬워졌다.
- 익명클래스를 처음 봐서 생소하다보니 헷갈렸으나, 자주 개념을 확인하니깐 덜 헷갈렸다.
