# 2026-06-08 인스턴스와 클래스

## 오늘 배운 내용

- Java에서 println은 C언어에서 \n와 같이 줄을 바꾸는 기능을 한다.
- Java에서 스트링 배열은 String과 같이 첫 문자를 대문자로 나타낸다.
- C언어는 절차 지향 언어이고, Java는 객체 지향 언어이다.
- C언어에서 상수는 const이고, Java에서는 final이고 값이 불변이다.
- Java에서 필드, 멤버 변수, 전역 변수는 다 같은 뜻을 나타내는 단어이다.
- 메모리와 하드디스크 그리고 SSD는 저장 공간이라는 공통점이 있다.
- 차이점은 메모리는 휘발성인 특징이 있고, 속도는 빠르지만 비싸다는 특징이 있다.
- 반면에 하드디스크와 SSD는 비휘발성인 특징이 있다.
- 오브젝트(object)는 현실 세계의 모든 객체이다.
- 클래스(class)는 오브젝트를 가상세계 용으로 구체화 한 것으로, 예를 들면 붕어빵 틀이나 설계도라고 비유할 수 있다.
- 인스턴스(instance)는 클래스를 활용해 메모리 상에 만들어 낸 것으로, 예를 들면 붕어빵이나 설계도를 보고 만든 실제 제품이라고 할 수 있다.
- 클래스명은 명사로 나타내고, 단어의 맨 처음은 대문자를 사용한다.
- 필드명은 명사로 나타내고, 최초 이외의 단어의 맨 처음은 대문자를 사용한다.
- 메서드명은 동사로 나타내고, 최초 이외의 단어의 맨 처음은 대문자를 사용한다.
- C언어의 함수는 파일 어디서나 독립적으로 존재 가능하다.
- 반면에 Java의 메서드는 반드시 class 내부에만 존재해야 한다.
- 정의한 클래스로 인스턴스를 생성할 수 있다.
- 이 클래스로 생성한 인스턴스를 넣을 수 있는 새로운 변수의 타입이 이용 가능해진다.
- 클래스를 정의하면 Java에서 이용가능한 타입의 종류가 늘어난다.
- 클래스에는 속성을 필드로, 동작을 메서드로 선언한다.
- this는 자기 자신의 인스턴스를 표시하는 키워드로, 생략가능하다.
- 어떤 클래스 타입 변수에 인스턴스가 담겨 있을 때 "변수명.필드명"이나 "변수명.메서드명()"으로 그 인스턴스의 필드나 메서드를 이용할 수 있다.

## 기억할 것

- Java에서 스트링 배열은 String과 같이 첫 문자를 대문자로 나타내는 것을 주의해서 기억한다.
- C언어와 Java의 차이점을 기억한다.
- 함수와 메서드의 차이점을 기억한다.
- 클래스와 필드 그리고 메서드 명명 규칙을 기억한다.
- 오브젝트와 클래스 그리고 인스턴스 간의 관계를 잘 기억한다.

## 실습 코드

```java
import java.util.Random;

public class Cleric {
    public static final int MAX_HP = 50;
    public static final int MAX_MP = 10;

    String name;
    int hp;
    int mp;

    void selfAid() {
        if (mp >= 5) {
            mp -= 5;
            hp = MAX_HP;
        }
    }

    int pray(int sec) {
        Random random = new Random();
        int recovery = sec + random.nextInt(3);

        int actual = Math.min(recovery, MAX_MP - mp);

        mp += actual;
        return actual;
    }

    public static void main(String[] args) {
        Cleric myCleric = new Cleric();

        myCleric.hp = 50;
        myCleric.mp = 10;
    }
}
```

## 어려웠던 점

- 스트링 배열을 나타낼 때, 맨 앞의 문자를 대문자를 사용하는 이유를 정확히 알지 못했다.
- 클래스와 필드 그리고 메서드 명명 규칙이 익숙해지는데 어려웠다.
- 메모리와 하드디스크 그리고 SSD의 특징을 정확히 알지 못했었다.

## 해결 방법

- 스트링 배열과 같이 미리 만들어둔 클래스는 맨 앞의 문자를 대문자로 표현하는 것을 확인했다.
- 클래스와 필드 그리고 메서드가 나타내는 특성을 이해한 후, 명명규칙을 보니 이해가 잘 됐다.
- 메모리와 하드디스크 그리고 SSD의 특징을 찾아보고 잘 이해하게 되었다.
